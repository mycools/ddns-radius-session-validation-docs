1. วัตถุประสงค์

ปรับปรุงระบบ DDNS เพื่อแก้ปัญหา Radius Accounting ส่งข้อมูลไม่เป็นลำดับ FIFO โดยเฉพาะกรณี Router ทำงานผิดปกติและเกิด Session Change ทำให้ระบบได้รับ Start Session ใหม่ มาก่อน Stop Session เก่า

ผลกระทบปัจจุบันคือระบบ DDNS อาจเข้าใจผิดว่า User Offline และ Clear CPE IP ผิด Session

2. ปัญหาปัจจุบัน

ปัจจุบันระบบใช้ Logic จาก Radius Accounting Packet โดยตรง:

Start -> Online
Stop  -> Offline + Clear CPE IP

แต่ในบางกรณี Radius ส่ง Packet ไม่เรียงลำดับ เช่น:

Start Session B
Stop Session A
  • Session B กำลัง Active อยู่
  • แต่ Stop ของ Session A ทำให้ระบบ Clear CPE IP
  • User ถูกมองว่า Offline ผิดพลาด

4. แนวทางแก้ไข

เพิ่มระบบจัดการ Active Session โดยสร้าง Table ใหม่ชื่อ:

RadiusAccountingSessions

Table นี้ใช้เก็บ Current Active Session ของแต่ละลูกค้า และใช้ตรวจสอบว่า Packet ที่เข้ามาเป็นของ Active Session ปัจจุบันหรือไม่ ก่อนจะ Update DDNS หรือ Clear CPE IP

5. Database Design

CREATE TABLE RadiusAccountingSessions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    username VARCHAR(255) NOT NULL,
    session_id VARCHAR(255) NOT NULL,

    cpe_ip VARCHAR(64),
    user_mac VARCHAR(64),

    nas_ip VARCHAR(64),
    nas_identifier VARCHAR(255),

    status ENUM('Start','Interim-Update','Stop') NOT NULL,

    started_at DATETIME NULL,
    last_interim_at DATETIME NULL,
    stopped_at DATETIME NULL,

    is_active BOOLEAN NOT NULL DEFAULT TRUE,

    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_username (username),
    INDEX idx_session_id (session_id),
    INDEX idx_active (username, is_active)
);

6. Workflow Logic

Start

  1. Receive Radius Start
  2. Insert/Update Active Session
  3. Update DDNS CPE IP จาก Framed-IP-Address
  4. Handle Duplicate CPE IP

Interim-Update

  1. Find Active Session
  2. Compare Session ID
  3. If match: update last_update
  4. If not match: Noop

Stop

  1. Find Active Session
  2. Compare Session ID
  3. If match: stamp Stop + clear CPE IP
  4. If not match: Noop

8. Deployment Strategy

Phase 1 — Collect Session IDs

เริ่มเก็บ Session ID ก่อน โดยยังไม่เปิดใช้งาน Validation Logic เป็นระยะเวลาประมาณ 2 สัปดาห์

ENABLE_RADIUS_SESSION_VALIDATION=false
ENABLE_CLEAR_DUPLICATE_CPE_ON_START=false
  1. Deploy Database Table ใหม่
  2. Deploy Radius Accounting Service ทีละเครื่อง
  3. Config A10 ให้ส่ง Accounting ไปเครื่องอื่นก่อน Deploy
  4. Verify Health Check และ Logs
  5. Rotate Traffic กลับ

Phase 2 — Enable Session Validation

เปิดใช้งาน Active Session Validation Logic และ Duplicate CPE Clear Logic

ENABLE_RADIUS_SESSION_VALIDATION=true
ENABLE_CLEAR_DUPLICATE_CPE_ON_START=true
  1. Deploy เฉพาะ Radius Accounting Service
  2. Config A10 ให้ส่ง Traffic ไปเครื่องอื่น
  3. Deploy Service Version ใหม่
  4. Verify Logs และ Validation Result
  5. Rotate Traffic กลับทีละเครื่อง

13. Node.js TypeScript Test Script

Script นี้ใช้จำลอง Radius Accounting Events เพื่อทดสอบ Session Validation Logic

type AccountingType = "Start" | "Stop" | "Interim-Update";

interface RadiusPacket {
  username: string;
  sessionId: string;
  cpeIp: string;
  type: AccountingType;
}

interface SessionRecord {
  username: string;
  sessionId: string;
  cpeIp: string;
  status: AccountingType;
  isActive: boolean;
}

const sessions = new Map<string, SessionRecord>();

function getActiveSession(username: string) {
  return [...sessions.values()].find(
    s => s.username === username && s.isActive
  );
}

function handlePacket(packet: RadiusPacket) {
  console.log(`\n${packet.type} ${packet.sessionId}`);

  if (packet.type === "Start") {
    for (const s of sessions.values()) {
      if (s.username === packet.username && s.isActive) {
        s.isActive = false;
      }
    }

    sessions.set(packet.sessionId, {
      username: packet.username,
      sessionId: packet.sessionId,
      cpeIp: packet.cpeIp,
      status: "Start",
      isActive: true,
    });

    console.log("Start Accepted");
    return;
  }

  const active = getActiveSession(packet.username);

  if (!active || active.sessionId !== packet.sessionId) {
    console.log("NOOP");
    return;
  }

  if (packet.type === "Interim-Update") {
    console.log("Interim Accepted");
    return;
  }

  if (packet.type === "Stop") {
    active.isActive = false;
    console.log("Stop Accepted");
  }
}

const username = "9612138730@fiberhome";

handlePacket({ username, sessionId: "SESSION_A", cpeIp: "10.181.6.70", type: "Start" });
handlePacket({ username, sessionId: "SESSION_A", cpeIp: "10.181.6.70", type: "Interim-Update" });
handlePacket({ username, sessionId: "SESSION_B", cpeIp: "10.181.6.70", type: "Start" });
handlePacket({ username, sessionId: "SESSION_A", cpeIp: "10.181.6.70", type: "Stop" });
handlePacket({ username, sessionId: "SESSION_B", cpeIp: "10.181.6.70", type: "Interim-Update" });
handlePacket({ username, sessionId: "SESSION_B", cpeIp: "10.181.6.70", type: "Stop" });
Run:
npx tsx radius-session-test.ts

Expected:
- Stop SESSION_A ต้อง Noop
- DDNS CPE IP ต้องไม่ถูก Clear
- Interim SESSION_B ต้อง Update ได้
- Stop SESSION_B เท่านั้นที่ Clear CPE IP

14. Expected Result

  • รองรับ Radius Packet Out-of-Order
  • ลด False Offline
  • ป้องกันการ Clear CPE IP ผิด Session
  • รองรับ Router Session Change ได้ถูกต้อง
  • ลดผลกระทบจาก Packet Delay / Packet Reorder
  • เพิ่มความเสถียรของ DDNS และ Port Forward System
